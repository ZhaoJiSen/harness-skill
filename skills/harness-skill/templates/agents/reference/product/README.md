# Product Features

> Index of the per-domain feature docs. Each file lists that domain's feature points, key flows,
> and the routes/UI behind them. Deep layer — the root `AGENTS.md` links here. "Product" means the
> product's functionality; it is **not** about production/deployment (see `deploy/` for that).
>
> Split rule: use a single `../features.md` for a small project; use this folder (one file per
> domain) once there are many domains or many feature points.
